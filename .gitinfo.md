## Useful Git Commands

### View History
```bash
git log --oneline                    # One-line log
git log --graph --all                # Graph view
git show <commit-hash>               # Show specific commit
```

### Create Branches for Phases
```bash
git branch phase-2-sellers
git checkout phase-2-sellers
# ... work on Phase 2

# When done, commit to master
git checkout master
git merge phase-2-sellers
```

### Staging Changes
```bash
git status                           # See what changed
git add .                            # Stage all changes
git add <file>                       # Stage specific file
git commit -m "message"              # Commit with message
```

### Pushing (When Ready)
```bash
# Set remote (GitHub example)
git remote add origin https://github.com/username/vintage_shop.git
git branch -M main                   # Rename to main if desired
git push -u origin master            # Push to remote
```

## Committing Best Practices

### Good Commit Messages
```
# Good
"Add seller onboarding forms and validation"
"Fix email verification token expiration bug"
"Refactor product image upload handler"

# Not Good
"Updated stuff"
"bug fix"
"more changes"
```

### Before Committing
```bash
# Always check what you're committing
git diff                             # See exact changes
git status                           # See files

# Stage incrementally
git add <specific-file>              # Stage one file
git commit -m "message"              # Commit
```
